# 12 — Dinamiskā programmēšana (DP) pamati

> DP olimpiādēs ir liels “game changer”. Mērķis: saprast ideju, nevis iemācīties 100 formulas.

## Kad domāt par DP
- “maksimums/minimums”, “veidu skaits”
- pārklājošas apakšproblēmas
- izvēles: ņemt/neņemt, sadalīt, iet pa režģi

## 2 pieejas
1) **Memoizācija (top-down)**: rekursija + `memo`  
2) **Tabulēšana (bottom-up)**: aizpildi masīvu no mazā uz lielo

## Šablons (memo)
```py
from functools import lru_cache

@lru_cache(None)
def dp(i):
    if i == 0:
        return 0
    # atgriez atbildi priekš i (no mazākiem i)
```

## “DP 1” uzdevumi (obligātie)
1. Climbing Stairs
2. House Robber
3. Longest Increasing Subsequence (LIS) — sākumā O(n²)
